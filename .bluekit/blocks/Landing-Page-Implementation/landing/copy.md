## How It Works

- **Section eyebrow:** How it works
- **Section heading:** Made for the way you actually work
- **Section subtitle:** No onboarding. No cloud setup. Just open it and go.

### Steps

**01 — Point it at a project**
Open any project folder. BlueKit attaches to it instantly. No setup, no config files required.

**02 — Build your context**
Create plans with milestones, track tasks, group reusable code blocks, and bookmark critical files.

**03 — Everything stays local**
Your SQLite database lives at `~/.bluekit`. No cloud, no accounts, no data leaving your machine.

**04 — Knowledge travels with your code**
Block folders live inside `.bluekit/` so your reusable patterns move with the repo.
